## Why

The repository currently has an Octave/Golden-model SFU reference and VHDL hardware, but no standalone C model for fast software-side regression. A C model is needed to reproduce the SFU datapath with Octave-generated LUTs and to support large txt-based comparison tests.

## What Changes

- Add a `cmodel/` tree containing a C implementation of the SFU function model.
- Use `Description/SFU_13` as the hardware behavior reference for range reduction, interpolation, normalization, and special-case handling.
- Load LUT files generated by the existing Octave flow instead of embedding duplicate coefficient tables.
- Add txt-based tests under `cmodel/tests`, including deterministic regression cases and generated bulk data.
- Add build/test commands for compiling the cmodel, generating test data, and comparing against Octave results.

## Capabilities

### New Capabilities
- `sfu-cmodel`: Standalone C model for SFU selector evaluation using generated LUT tables and txt-based regression tests.

### Modified Capabilities

## Impact

- Adds `cmodel/src`, `cmodel/tests`, and `cmodel/Makefile`.
- Reuses existing Octave LUT generation and golden-model scripts.
- Does not modify VHDL sources or existing LUT generation behavior.
